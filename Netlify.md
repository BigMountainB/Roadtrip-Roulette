# Deploying DUI to Netlify

Setup is already done. **Pushing a commit to `origin/main` triggers an auto-deploy** — no Netlify CLI, no manual build needed. The whole workflow is supposed to be 4 commands. If it takes longer than that, something's wrong; check the **Traps** section below.

## How to deploy (4 commands, ~30 seconds)

```bash
cd /Users/brendanbaughn/Documents/Claude/DUI

# 1. Stage ONLY runtime-relevant changes — never `git add -A` blindly
git add src public PROJECT_OVERVIEW.md

# 2. Commit with the Netlify-recognized author email (see Trap #1)
git commit --author="Brendan Baughn <brendanbaughn@gmail.com>" \
  -m "Short description of what changed"

# 3. Push — Netlify auto-builds from origin/main
git push origin main

# 4. (Optional) watch the build land at:
#    https://app.netlify.com/teams/brendanbaughn/builds
```

Build typically completes in 15-25 seconds. Production URL serves the new bundle immediately after.

## Traps (each of these has bitten us before)

### Trap #1 — "Unrecognized Git contributor" → build fails silently

Netlify's plan allows one contributor on private repos. That contributor email is **`brendanbaughn@gmail.com`**. If the commit author is anything else, Netlify rejects the build with `"One or more builds failed due to an unrecognized Git contributor"`.

Local git default may be `nwbproperty@gmail.com` (or whatever). **Always pass `--author="Brendan Baughn <brendanbaughn@gmail.com>"` to `git commit`.** Don't change `git config user.email` globally — the override flag is cleaner.

### Trap #2 — `git add -A` includes the Images/ folder → push dies with `pack-objects signal 10`

`Images/` at the repo root is a **local-only archive** for unused/source art (PSDs, alternate variants, badge originals). It holds **>50 MB of PSDs** that should NEVER ship. Committing them makes `git pack-objects` SIGBUS during push.

- Never `git add -A`. Stage specific folders: `git add src public PROJECT_OVERVIEW.md`.
- If you ever see `pack-objects died of signal 10`, this is the cause — unstage Images/: `git reset HEAD Images/`, then commit and push again.
- Want to permanently prevent it? Add `Images/` to `.gitignore` (but the folder is already partially tracked in history, so files added before that point stay tracked unless `git rm --cached -r Images/` is also run).

### Trap #3 — Netlify pulls from GitHub, not from your laptop

The deploy is wired to GitHub (`https://github.com/BigMountainB/DUI`). Local file changes alone do nothing. You must `git push origin main` for the deploy to fire. If you're staring at the Netlify dashboard waiting for changes to appear, the question is always "did the push succeed?"

### Trap #4 — `dist/` is build output, not source

Don't ever edit files under `dist/` directly. Don't commit `dist/`. Netlify runs `npm run build` itself; your local `dist/` is throwaway and is already in `.gitignore`.

### Trap #5 — `ios/App/App/public/` is the Capacitor sync target

Same deal. Don't edit; regenerated by `npx cap sync ios`. Already gitignored.

## Configuration (already set, just for reference)

- **Build command:** `npm run build` (in `netlify.toml`)
- **Publish directory:** `dist/`
- **Node version:** `18` (Node 20+ hits an npm-10 optional-deps bug that breaks rollup native binaries during Netlify's build)
- **Site name on Netlify:** `drivedrunk` (URL is `drivedrunk.netlify.app` or whatever custom domain you've wired)
- **GitHub remote:** `origin = https://github.com/BigMountainB/DUI.git`

## When things look broken

| Symptom | Likely cause | Fix |
|---|---|---|
| Push hangs / `pack-objects signal 10` | Images/ files staged | `git reset HEAD Images/`, retry |
| Build fails on Netlify with "unrecognized contributor" | Wrong author email | Use `--author=` flag; redo commit |
| Netlify dashboard shows no recent build | Push didn't succeed | Check `git push origin main` output; verify with `git fetch && git log origin/main -1` |
| Build succeeds but old code in production | Browser cache | Hard-refresh (Cmd+Shift+R) |
| Build fails with rollup native-binary error | Node version drift | Confirm `netlify.toml` has `NODE_VERSION = "18"` |
